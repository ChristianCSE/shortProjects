    FB Likes Association [Independent]

    March 2016 - Present Console App Programmer
    Java, PHP, Python, FB Graph API

    Description: Once given a text file, the program asks the user to input a FB Like. This is to see if a FB Like already exists in the Trie. Once the console receives input from the user, the console will tell you if the Like already exists in your text file. But that is an extremely boring feature, since you could simply do this through a hash map, array, etc. What's the point?
    Purpose: I made this so I can eventually use it to gather every possible Like on Facebook. Not only that, but I also wanted to be able to store every Likable Page ID. In other words, certain pages cannot be Liked. For instance, if you like my status, FB's API will not consider you Liking my status as a Like, hence it will not give you my profile's ID (id=100011117437562).
    2nd Purpose: As I build this data set, I also want to associate similar Likes with one another. For example, I would want to consider any Like associated with Facebook under the same umbrella term: "Facebook" == (these belong to it) ==> "Facebook Jobs", "Facebook Data Science", "Facebook Hacker Cup", etc. As of now, I am using a Trie to do this, but I believe there is a more efficient way.
